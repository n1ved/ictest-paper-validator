# Extraction and Processing

Before validation can occur, the raw PDF file must be parsed into a structured, readable format. This is handled by the files in the `app/processors/` directory.

## `extractor.py`

This module relies heavily on `PyMuPDF` (`fitz`) to extract comprehensive data from the PDF. The main function, `comprehensive_pdf_extraction(pdf_path)`, calls several helper functions to build a complete JSON representation of the document:

- **Metadata Extraction (`extract_metadata`)**: Retrieves document properties like author, creator, page count, and encryption status. This is crucial for verifying if the document was generated by specific software (e.g., PDFeXpress).
- **Raw Text Extraction (`extract_text_from_pdf`)**: Extracts the plain text content page by page.
- **Formatted Text Extraction (`extract_text_with_formatting`)**: **This is the core of the validation engine.** It uses `page.get_text("dict")` to extract text blocks. Each block contains lines, and each line contains "spans". A span is a continuous string of text with identical formatting (font, size, color, flags like bold/italic, and its physical bounding box `bbox`). This allows the checkers to know exactly *where* text is and *how* it looks.
- **Font Information (`extract_fonts_info`)**: Gathers a list of all fonts embedded and referenced in the document.
- **Tables (`extract_tables_from_pdf`)**: Attempts to locate and extract structured tabular data from the document pages using PyMuPDF's table finding capabilities.

The resulting JSON object is used as the single source of truth for all subsequent checkers.

## `text_processor.py`

The `PDFTextProcessor` class acts as a utility wrapper around the extracted JSON data, making it easier for the checkers to query the document. 

While the checkers often access the JSON data directly, `PDFTextProcessor` provides structured methods to:
- Load the formatted text data.
- Flatten the nested structure (blocks -> lines -> spans) into a linear list of `text_segments`.
- Map flag bits to boolean properties (`is_bold`, `is_italic`, `is_superscript`, `is_subscript`).
- Create a `word_index` for fast searching of specific words.
- Provide querying functions like `search(query)`, `search_word(word)`, and `filter_by_properties(**filters)` to easily find text matching specific criteria (e.g., finding all text that is 14pt and Bold).
- Calculate summaries and distributions of fonts and sizes across the document.
